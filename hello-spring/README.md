# 스프링 입문 - 코드로 배우는 스프링 부트, 웹 MVC, DB 접근 기술

## 🚀 비즈니스 요구사항 정리
- 데이터 : 회원 ID, 이름
- 기능 : 회원 등록, 조회
- 아직 DB가 선정되지 않음(가상의 시나리오)
  - 우선 인터페이스로 구현 클래스를 변경할 수 있도록 설계 (MemberRepository)
  - 초기 개발 단계에서는 구현체로 가벼운 메모리 기반의 데이터 저장소 사용 (MemoryMemberRepository)
    - 저장 기술 예) JDBC, MyBatis, JPA 등

## 🎯 프로그래밍 요구사항
- 회원 도메인과 리포지토리 만들기
- 회원 리포지토리 테스트 케이스 작성
- 회원 서비스 개발
- 회원 서비스 테스트
